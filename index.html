Welcome

Castor, a small solid Web-Framework written in PHP.

The build in classes couldt give a high scalable background for your own MVC or whatever concept. A xml-configuration couldt describe a complete homepage and works powerfull with XSLT-Templates. Serveral additions to add constants and individual modules, prepares a data-model for a page.

Use case

As the productive base, a class named as "xsltDocument" loads controller classes that extend Application Objects. Data, prepared by modules and "document elements", feeds controller classes with elements given for a action. The data as array, DomDocument object or plaintext, couldt served and adopt in serveral data formats like xml and json. A little copy paste of the action configuration, transforms the hole base code from a dynamic page to an asynchronous service.

The Goal

The idea, to describe a page with a xml config-file is allready the goal of the project. A plugin system shouldt extend tags to describe simple interactions for and from the view. Also system plugins shouldt extend the baseline of the system. The framework will equipped with some simple modules for typical jobs of PHP-Engineers.
